## 二、前端配置详解

### 2.1 登录页面配置 (`ruoyi-ui/src/views/login.vue`)

#### 注册开关配置（第97行）

```javascript
// 注册开关 - 控制是否显示"立即注册"链接
register: true,  // true=显示注册入口, false=隐藏注册入口
```

**作用说明**：
- 当 `register: true` 时，登录页面会显示"立即注册"链接
- 点击后跳转到 `/register` 路由
- 该配置仅控制前端显示，真正的注册功能开关在**后端数据库配置**中

#### 登录表单核心配置

```javascript
loginForm: {
  username: '',  // 用户名
  password: '',  // 密码（前端加密后传输）
  code: '',      // 图形验证码
  uuid: ''       // 验证码唯一标识
}
```

#### 登录流程

1. 用户输入用户名/密码
2. 获取图形验证码（调用 `/captchaImage` 接口）
3. 前端对密码进行加密
4. 调用 `/login` 接口提交登录请求
5. 成功后保存 Token 到 Cookie/SessionStorage
6. 跳转到首页

## 四、核心业务流程

### 4.1 登录流程

```
前端请求                              后端处理
    |                                      |
    |-- POST /login ---------------------->|
    |   {username, password, code, uuid}   |
    |                                      |
    |                                      |-- 验证图形验证码
    |                                      |-- 查询用户信息
    |                                      |-- 验证密码（BCrypt匹配）
    |                                      |-- 生成JWT Token
    |                                      |-- 保存登录日志
    |                                      |
    |<-- 返回 Token -----------------------|
    |                                      |
    |-- 保存 Token 到 Cookie              |
    |-- 跳转首页                          |
```

### 4.2 注册流程

```
前端请求                              后端处理
    |                                      |
    |-- GET /sendEmailCode?email=xxx ---->|
    |                                      |
    |                                      |-- 生成6位验证码
    |                                      |-- 保存到Redis（有效期60秒）
    |                                      |-- 发送邮件
    |<-- 返回成功 --------------------------|
    |                                      |
    |-- POST /register ------------------->|
    |   {email, emailCode, username,      |
    |    password, confirmPassword}       |
    |                                      |
    |                                      |-- 验证邮箱验证码
    |                                      |-- 检查用户名是否重复
    |                                      |-- 密码加密（BCrypt）
    |                                      |-- 保存用户信息
    |<-- 返回成功 --------------------------|
    |                                      |
    |-- 跳转到登录页                      |
```

## 五、关键技术点

### 5.1 密码加密机制

**BCrypt 加密**（`SecurityUtils.encryptPassword()`）：

```java
public static String encryptPassword(String password) {
    return BCrypt.hashpw(password, BCrypt.gensalt());
}

// 验证密码
BCrypt.checkpw(rawPassword, encodedPassword)
```

**特点**：
- 单向加密，无法逆向解密
- 相同密码每次加密结果不同（加盐）
- 安全性高，抗暴力破解

### 5.2 Token 机制

**JWT（JSON Web Token）**：

```
Token 结构：Header.Payload.Signature

Header: {"alg": "HS256", "typ": "JWT"}
Payload: {"userId": 1, "username": "admin", "exp": 1717700000}
Signature: HMACSHA256(base64UrlEncode(header) + "." + base64UrlEncode(payload), secret)
```

**Token 验证流程**：
1. 前端请求携带 `Authorization: Bearer xxx`
2. 后端过滤器解析 Token
3. 验证签名有效性
4. 从 Payload 中提取用户信息
5. 设置 SecurityContext

### 5.3 验证码机制

**图形验证码**（Kaptcha）：
- 生成随机字符 + 干扰线
- 验证码存储到 Redis（key=uuid，有效期5分钟）
- 验证时对比用户输入与 Redis 存储值

**邮箱验证码**：
- 生成6位数字
- 存储到 Redis（key=email_codes:xxx@xxx.com，有效期60秒）
- 通过 Spring Mail 发送邮件

## 八、常见问题排查

### 8.1 数据库连接失败

**现象**：启动时提示 `Access denied for user 'root'@'localhost'`

**解决**：
1. 检查 MySQL 服务是否启动
2. 确认用户名密码正确
3. 确认数据库已创建

### 8.2 注册功能不可用

**现象**：提示"当前系统没有开启注册功能"

**解决**：
```sql
UPDATE sys_config SET config_value = 'true' WHERE config_key = 'sys.account.registerUser';
```

### 8.3 邮箱验证码发送失败

**现象**：提示"发送邮件失败"

**解决**：
1. 检查邮箱配置是否正确
2. 确认授权码已正确获取
3. 检查网络是否可访问邮箱服务器

### 8.4 端口占用

**现象**：`Address already in use`

**解决**：
```bash
# 查找占用进程
netstat -ano | findstr ":8080"

# 终止进程
taskkill /F /PID <进程ID>
```
